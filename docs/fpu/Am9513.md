# Am9513 (P2) - Async Scalar IEEE Tier

This document summarizes the Project Carbon **Am9513 P2** personality as exposed
by the shared Am95xx accelerator block.

## Status and identity

- Presented tier: **P2 (Am9513)**.
- Feature bits: `AM9513_ASYNC_SCALAR`, `AM9512_IEEE_PLUS` (plus `AM9514_VECTOR` /
  `AM9515_TENSOR` when higher tiers are enabled).
- The same hardware block hosts **P0 (Am9511)** and **P1 (Am9512-plus)** legacy
  personalities via the legacy CSR window.
- P2 and above are **CAI-native**; legacy stack accesses are P0/P1 only.

## Implemented scope (v1)

- CAI opcode group: `CARBON_AM95_SCALAR`.
- Formats: `CARBON_FMT_BINARY16`, `CARBON_FMT_BFLOAT16`, `CARBON_FMT_BINARY32`,
  `CARBON_FMT_BINARY64`.
- Scalar ops: add/sub/mul/div/sqrt/fma, min/max/cmp/class, conversions, and a
  minimal set of transcendental functions (sin/cos/sincos/exp/log/pow/atan2/hypot).
- Integer conversions: `i32<->f32`, `i64<->f64`.
- Per-context rounding mode and sticky IEEE flags (NV/DZ/OF/UF/NX).
- Register-file fast path (P2 only): result-to-reg writes `F0..F15`.

## Determinism and ordering

- CAI descriptors are executed **in order** with a single-outstanding fabric
  micro-engine.
- I/O and memory are accessed through ordered fabric attributes; no speculative
  side effects are permitted.

## Deferred / TODO

- Decimal floating-point formats.
- Full IEEE 754 edge-case compliance and exception traps/masks.
- Wider CAI opgroups (vector/tensor live in P3/P4).
- Multi-issue CAI fetch and deeper pipelining.
